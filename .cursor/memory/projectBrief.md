# Project Brief

## プロジェクト名
MoodTune - 季節×天気×時間でプレイリスト提案

## 核心的なゴール
天気と時間に応じて動的に変化する背景色とアニメーションを備えた、音楽プレイリスト探索アプリケーション。ユーザーの現在の環境（天気、時間帯）に合わせて、適切な音楽プレイリストを視覚的に提案する。

## 主要要件

### 機能要件
1. **天気連動システム**
   - 位置情報から現在の天気を取得（OpenWeatherMap API）
   - 14種類の天気タイプに対応（Clear, Clouds, Rain, Drizzle, Thunderstorm, Snow, Mist, Fog, Haze, Dust, Sand, Ash, Squall, Tornado）
   - 時間帯（朝/昼/夕方/夜）に応じた背景色の動的変化

2. **視覚的フィードバック**
   - 天気と時間帯に応じたグラデーション背景（4色: top, from, via, to）
   - 各天気タイプに応じたアニメーション（雨、雪、雲、霧など）
   - 天気アイコンのテーマカラー表示

3. **プレイリスト探索UI**
   - レコード盤風のインタラクティブUI
   - スワイプ/ドラッグでプレイリストを切り替え
   - 5つのプレイリスト（チルアウト、アンビエント、クラシック、ローファイヒップホップ、アコースティック）

4. **開発・テスト機能**
   - メインUI上にテストパネル（右上の設定アイコン）
   - 天気タイプと時間帯を手動で設定可能
   - 選択時に即座に反映（適用ボタン不要）
   - リセットで実際の天気に戻る

### 非機能要件
- レスポンシブデザイン（モバイル・デスクトップ対応）
- パフォーマンス最適化（アニメーションの軽量化）
- アクセシビリティ（キーボード操作、スクリーンリーダー対応）

## 技術的制約
- Next.js 15 App Router必須
- TypeScript必須
- shadcn/uiコンポーネントライブラリ使用
- Tailwind CSSのみでスタイリング（カスタムCSS最小限）
